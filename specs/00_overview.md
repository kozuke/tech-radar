# 情報収集アプリ 仕様書（概要）

## 1. 背景と課題
エンジニアとして日々大量の技術情報（ブログ、公式リリース、ニュース、Qiita等）に触れる必要があるが、
以下の課題がある。

- 情報源が分散しており巡回コストが高い
- 全文を読む時間がない
- 「結局何が重要か」が分かりづらい
- 後から振り返れない（流れてしまう）

## 2. 本プロジェクトの目的
本プロジェクトは、
**技術情報の収集・要約・蓄積・閲覧を自動化し、意思決定と学習を高速化する**
ことを目的とする。

個人利用を前提とし、以下を重視する。

- 無料枠中心での運用
- 完全自動化（定期実行）
- データ資産が手元（Git）に残る
- UIと収集ロジックの分離

## 3. スコープ
### 対象
- IT / ソフトウェアエンジニア向け記事
- RSS / 検索ベースで取得可能な公開記事

### 非対象
- リアルタイム速報
- 有料記事全文の保存
- SNS的機能（コメント・いいね）

## 4. 全体像

```
[ GitHub Actions ]
  └─ 記事収集 → 本文抽出 → LLM要約 → data/ に保存 → commit

[ GitHub Pages ]
  └─ data/ を読み込み → 一覧・詳細を静的表示
```

## 5. 設計思想
- **ファイルはAPI**
- **Markdownは長期保存に強い**
- **UIは捨てられるがデータは捨てない**
